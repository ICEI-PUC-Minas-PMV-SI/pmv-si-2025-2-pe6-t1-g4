# Apresentação da Solução

Resumo da apresentação — TrainerHub
Apresentação:
- Projeto / equipe: TrainerHub — solução multiplataforma desenvolvida por Augusto, Leonardo, Lucas e Wellington para a disciplina de Arquitetura de Sistemas Distribuídos.
- Contexto: mercado fitness em forte digitalização (crescimento de academias e demanda por treinos, fichas e orientação online).
- Problema: processos manuais, sistemas fragmentados, perda de históricos, baixa automação e engajamento dos alunos.
- Solução: app mobile + painel web para digitalizar fichas, agendamentos, evolução física, comunicação e feedback em tempo real; objetivo de aumentar retenção e eficiência.
- Backend: API centralizada (RESTful), autenticação via Supabase Auth, backend em PHP com Symfony, banco Supabase — foco em sincronização, escalabilidade e segurança.
- Front-end Web: interface responsiva para personal trainers/gestores — funcionalidades de cadastro de aluno, visualização de treinos, acompanhamento de evolução e agendamento; tecnologia principal: React.js; estilo visual: paleta escura com destaques em vermelho.
- Front-end Mobile: app para alunos com cadastro, visualização de treinos e agendamentos/remarcações; implementado em React Native + Expo; design em glassmorphism com tons de azul/roxo/vernmelho e acabamento “premium”.
- Wireframes e modelagem: apresentados para suportar as telas e fluxos do produto.
- Conclusão: solução técnica coerente e simples de manter, alinhada às necessidades do ambiente fitness — produto funcional e intuitivo.
